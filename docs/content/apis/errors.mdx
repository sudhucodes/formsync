# API Errors

FormSync uses standard HTTP status codes and a consistent error format.

---

## Error Format

```json
{
  "success": false,
  "message": "Error message"
}
```

---

## Common Errors

### Missing API Key

**Status:** `401 Unauthorized`

```json
{
  "success": false,
  "message": "API key required"
}
```

---

### Invalid API Key

**Status:** `401 Unauthorized`

```json
{
  "success": false,
  "message": "Invalid API key"
}
```

---

### Form Not Found

**Status:** `404 Not Found`

```json
{
  "success": false,
  "message": "Form not found"
}
```

---

### Missing Form ID

**Status:** `400 Bad Request`

```json
{
  "success": false,
  "message": "Form ID is required"
}
```

---

### Rate Limit Exceeded

**Status:** `429 Too Many Requests`

```json
{
  "success": false,
  "message": "Rate limit exceeded"
}
```

---

### Server Error

**Status:** `500 Internal Server Error`

```json
{
  "success": false,
  "message": "Something went wrong"
}
```

---

## Need Help?

If you encounter unexpected errors, please contact support or open an issue on [GitHub](https://github.com/sudhucodes/formsync).
